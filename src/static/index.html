                         我的订阅链接
                    

                    
每次在"编排工作区"保存配置后,系统会自动生成对应订阅。请保护好这些链接,任何持有者都能拉取你的代理列表。

                


                
                

                    
                    

                        
尚未生成订阅

                        
请先在"编排工作区"导入节点并设计分组,然后点击下方"保存配置并生成订阅"按钮。

                    

                


                
                

                    

                        
                        

                            
                            
                            
                        

                    

                    

                        
                        

                            
                            
                            
                        

                    

                    

                        
                        

                            
                        

                    

                    

                        
 订阅 token 由系统生成,与登录 UUID 分离,可在保存时点"重置 token"使旧链接失效。

                        
外部 API: GET /api/v1/config,Header 带 Authorization: Bearer <你的登录UUID>

                    

                


                

                    
                    
                

            


            
            

                

                    

                         外部 API 速查
                    

                    
供外部脚本/客户端读写配置

                

                

                    
整体配置

                    

                        
GET /api/v1/config

                        
取自己的工作区 JSON(含 nodes/groups/订阅链接)

                    

                    

                        
PUT /api/v1/config

                        
body = {"nodes":[...],"groups":[...],"busNames":{...}};整体覆盖,自动重新生成订阅

                    

                    

                        
GET /api/v1/subscriptions

                        
返回 clash/v2ray/分组 订阅 URL 清单

                    


                    
节点维度 (CRUD)

                    

                        
GET /api/v1/nodes

                        
列出全部节点

                    

                    

                        
GET /api/v1/nodes/:nodeId

                        
查询单个节点

                    

                    

                        
POST /api/v1/nodes

                        
新增节点;body 必填 name/type/server/port + 协议字段;可选 groupIds 一并加入

                    

                    

                        
PUT /api/v1/nodes/:nodeId

                        
更新节点字段(只传需要改的字段即可)

                    

                    

                        
DELETE /api/v1/nodes/:nodeId

                        
删除节点(自动从所有分组中移除)

                    


                    
分组维度 (CRUD)

                    

                        
GET /api/v1/groups

                        
列出所有分组

                    

                    

                        
GET /api/v1/groups/:groupId

                        
查询单个分组

                    

                    

                        
POST /api/v1/groups

                        
新增分组;body 必填 name,可选 role(entry/common/exit/relay)、nodes 列表

                    

                    

                        
PUT /api/v1/groups/:groupId

                        
更新分组(可改名、改角色、改成员)

                    

                    

                        
DELETE /api/v1/groups/:groupId

                        
删除分组(节点本身保留;引用此组的 relay 会自动断开)

                    

                    

                        
POST /api/v1/groups/:groupId/nodes/:nodeId

                        
把节点加入分组

                    

                    

                        
DELETE /api/v1/groups/:groupId/nodes/:nodeId

                        
把节点移出分组(节点本身保留)

                    


                    

                        鉴权: Authorization: Bearer <登录 UUID>

                        Admin 在所有写入接口上可加 ?uuid=<目标用户> 操作他人配置。
